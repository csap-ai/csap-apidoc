## 🎯 核心功能实现

### 参数分组逻辑
```typescript
export const groupParametersByType = (params: Parameter[]): GroupedParams => {
  const grouped = {
    PATH: [],
    QUERY: [],
    BODY: [],
    FORM_DATA: [],
    HEADER: [],
    DEFAULT: []
  };

  params.forEach(param => {
    const type = param.paramType || 'DEFAULT';
    grouped[type].push(param);
  });

  return grouped;
};
```

### 请求构建逻辑
```typescript
export const buildRequestData = (params, jsonData) => {
  const grouped = groupParametersByType(params);
  
  return {
    pathParams: extractParams(grouped.PATH, jsonData),
    queryParams: extractParams([...grouped.QUERY, ...grouped.DEFAULT], jsonData),
    bodyData: extractBodyData(grouped.BODY, jsonData),
    formData: extractParams(grouped.FORM_DATA, jsonData),
    headers: extractParams(grouped.HEADER, jsonData)
  };
};
```

### PATH 参数替换
```typescript
export const replacePathParams = (url, pathParams) => {
  let result = url;
  Object.keys(pathParams).forEach(key => {
    result = result.replace(`{${key}}`, String(pathParams[key]));
    result = result.replace(`:${key}`, String(pathParams[key]));
  });
  return result;
};
```

## 📊 使用示例

### 示例 1: 路径参数
```json

// 参数: id (paramType: PATH)
```

```json
// API: POST /api/article/{id}/comment
// 参数:
//   - id (paramType: PATH)
//   - page (paramType: QUERY)
//   - content (paramType: BODY)
//   - token (paramType: HEADER)

  "page": 1,
  "content": "评论内容",
  "token": "abc123"
}

Headers: { token: "abc123" }
Body: { "content": "评论内容" }
```

## ⚙️ 配置说明

### 后端 API 数据格式

请求参数应包含 `paramType` 字段：

```json
{
  "request": [
    {
      "name": "id",
      "dataType": "int",
      "required": true,
      "paramType": "PATH",
      "description": "用户ID"
    },
    {
      "name": "page",
      "dataType": "int",
      "required": false,
      "paramType": "QUERY",
      "description": "页码"
    },
    {
      "name": "data",
      "dataType": "object",
      "required": true,
      "paramType": "BODY",
      "description": "数据对象"
    }
  ]
}
```

### 兼容性

如果参数没有 `paramType` 字段：
- 自动归类为 `DEFAULT`
- 等同于 `QUERY` 参数
- 不影响现有功能

## 📝 注意事项

1. **PATH 参数命名**
   - URL 占位符与参数名必须一致
   - 支持 `{param}` 和 `:param` 两种格式

2. **参数优先级**
   - POST/PUT/PATCH: 优先使用 BODY 作为请求体
   - GET/DELETE: 使用 QUERY 参数

3. **Form Data**
   - 自动设置 `Content-Type: multipart/form-data`
   - 适用于文件上传场景

4. **Header 参数**
   - 会添加到所有请求中
   - 避免与系统默认 headers 冲突
