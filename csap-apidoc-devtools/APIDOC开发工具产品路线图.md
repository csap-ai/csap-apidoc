## 产品定位

### 当前定位
API 接口文档管理和配置工具，服务于 Java Spring Boot 项目

### 未来定位
**面向 Java 开发者的一体化 API 开发效率平台**
- 不仅是文档工具，更是开发效率工具
- 深度集成 Spring Boot 生态
- 打通前后端协作链路

### 目标用户
1. **主要用户**: Java 后端开发者
2. **次要用户**: 前端开发者、测试工程师
3. **管理用户**: 技术 Leader、架构师

---

## 核心价值

### 用户痛点
1. ❌ 手动维护文档繁琐且容易遗漏
2. ❌ 接口变更通知不及时，前后端联调困难
3. ❌ 测试需要频繁切换工具（Postman/Swagger）
4. ❌ 字段配置重复劳动多
5. ❌ 缺少版本管理，接口变更难追溯

### 产品价值
1. ✅ **效率提升**: 自动化扫描 + 字段复用，减少 70% 重复工作
2. ✅ **协作增强**: 版本管理 + 变更通知，减少沟通成本
3. ✅ **一站式**: 配置、测试、文档生成，无需切换工具
4. ✅ **标准化**: 统一团队接口规范，提升代码质量

### 1.3 Mock 数据生成

#### Mock 规则示例
```javascript
字段类型映射:
- String → @cname (中文名) / @email / @url
- Integer → @integer(1, 100)
- Date → @datetime
- Boolean → @boolean
- 手机号 (验证规则) → @phone
- 身份证 (验证规则) → @id
```

#### 交互设计
在测试界面添加"生成 Mock 数据"按钮

#### 技术方案
- 集成 Mock.js 或自研
- 根据字段名智能推断（username → 姓名，email → 邮箱）
- 考虑验证规则约束

#### 验收标准
- [ ] 自动识别常见字段类型生成合理数据
- [ ] 支持自定义 Mock 规则
- [ ] 生成的数据符合验证规则

### 1.4 文档导出增强 🔥
**价值**: 提升文档流通性，支持多场景使用

#### 功能描述
1. **导出 OpenAPI 3.0 / Swagger 格式**
   - 标准 JSON/YAML 格式
   - 可直接导入 Swagger UI/Postman
   - 支持完整的参数、验证规则导出
   
2. **JSON Schema 导入导出** ⭐ NEW
   - 导出接口参数为 JSON Schema
   - 从 JSON Schema 导入生成参数配置
   - 自动映射验证规则（required, pattern, min/max）
   - 支持批量导入
   
3. **导出 Markdown 文档**
   - 美观的格式化文档
   - 支持导出单个接口或整个项目
   - 包含请求示例和响应示例

4. **生成 TypeScript 类型定义** 🔥
   - 根据返回参数生成 interface
   - 自动处理嵌套类型
   - 可配置命名规则

#### TypeScript 生成示例
```typescript
// 生成的代码示例
export interface ProductRequest {
  groupId: number;
  del: boolean;
  sales: number;
  paymentType?: string;
}

export interface ProductResponse {
  groupId: number;
  del: boolean;
  sales: number;
  store: {
    address: string;
    groupId: number;
    mobile: string;
  };
}
```

#### JSON Schema 映射规则
```javascript
// 参数类型映射
String → { "type": "string" }
Integer/Long → { "type": "integer" }
Boolean → { "type": "boolean" }
Array → { "type": "array", "items": {...} }
Object → { "type": "object", "properties": {...} }

// 验证规则映射
required: true → "required": ["fieldName"]
@Pattern(regex) → "pattern": "regex"
@Min(value) → "minimum": value
@Max(value) → "maximum": value
@Length(min, max) → "minLength": min, "maxLength": max
@Email → "format": "email"
```

#### 技术方案
- OpenAPI 规范对照实现
- JSON Schema 标准实现
- 模板引擎生成文档（Handlebars/Mustache）
- TypeScript AST 生成工具

#### 验收标准
- [ ] 导出的 OpenAPI 文档可在 Swagger UI 中正常展示
- [ ] JSON Schema 导出符合标准，验证规则完整
- [ ] 可从 JSON Schema 导入并正确生成参数配置
- [ ] Markdown 文档格式美观，包含完整信息
- [ ] TypeScript 类型定义准确，可直接使用
- [ ] 支持批量导入导出

   - 身份证脱敏：110***********1234
   - 邮箱脱敏：abc***@example.com

2. **脱敏场景**
   - 测试用例保存时自动脱敏

   - 标记字段为敏感（手机、身份证、密码等）

   - 自动识别建议

```

│ 预览: 130****5678                  │
└────────────────────────────────────┘
```

#### 技术方案
- 前端展示层脱敏

#### 验收标准

```java
// 限流规则配置
@SentinelResource(
```

## 风险与应对

### 技术风险
1. **实时协作性能问题**
   - 应对: WebSocket 优化，消息队列削峰
   
2. **大项目性能问题**
   - 应对: 分页加载，虚拟滚动，索引优化

3. **数据安全风险**
   - 应对: 加密存储，权限控制，审计日志

### 市场风险
1. **竞品压力（Swagger/Apifox）**
   - 应对: 差异化定位（深度集成 Spring Boot）
   
2. **付费意愿低**
   - 应对: 提供足够价值，长期免费试用

3. **推广困难**
   - 应对: 开源社区版，内容营销，KOL 合作
